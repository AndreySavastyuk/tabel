# Табель СКУД v0.3 — что нового и как пользоваться

Версия `legacy/SCUD(fixed_time)_v0.3.py` развивает `v0.2`. Старые листы
(«Выгрузка», «Фиксированное время», «Полная выгрузка») остаются **без
изменений** — добавлены новые возможности и аналитические листы.

## Новые модули (вынесены в `legacy/`; поддерживаемый аналог — пакет `engine/`)
- `legacy/model.py` — модель дня (`DayRecord`) и свод по сотруднику (`EmployeePeriod`).
- `legacy/refdata.py` — чтение справочников, авто-сопровождение и шаблоны.
- `legacy/compute.py` — вычисления (обед, опоздания, переработки, нормы, отклонения).
- `legacy/report.py` — писатели аналитических листов.

## Справочники в папке `ЛЭЗ/` (создаются автоматически как шаблоны)

| Файл | Что заполнить |
|---|---|
| **Справочник_сотрудников.xlsx** | Программа дописывает все ФИО сама. Вы заполняете: **Отдел**, **Кабинет** (для больших отделов — разбивка по кабинетам), **График**, **Контроль ЛЭЗ** (да/нет — проходит ли человек проходную). Фикс.время мигрирует из Сотрудники.txt. |
| **Графики_нормы.xlsx** | По каждому графику и месяцу (на год вперёд): **Норма** (часы), **Начало смены**, **Длит.смены**, **Обед нач**, **Обед кон**. Пример-строку `ПРИМЕР_5x2` удалите. |
| **Отсутствия.xlsx** | Выгрузка бухгалтерии: ФИО, Тип (отпуск/больничный), Дата с, Дата по. |
| **Командировки.xlsx** | Заполняете вы: ФИО, Дата с, Дата по. |
| **Настройки.json** | Пороги отклонений (можно не трогать). |

Пока графики/отделы не заполнены — табель работает как раньше, аналитические
листы показывают «нет нормы» и не делятся по отделам. После заполнения —
листы автоматически наполняются.

> **Пошаговая инструкция по заполнению — в файле `НАСТРОЙКА.md`** (отделы,
> обед, графики, нормы, отсутствия, пороги — как настроить на всех сотрудников).

## Что считается
- **Ночные/сменные смены** — события разбиваются на смены по времени (перерыв
  больше `shift_gap_min`, по умолч. 5 ч, начинает новую смену). Поэтому смена
  «вход 20:00 → выход 07:30 следующего дня» считается как ~11.5 ч, а не как
  ноль/минус. Внутрисменный обед/короткая отлучка сливаются в одну смену.
  Пороги — в `ЛЭЗ/Настройки.json` (`shift_gap_min`, `max_shift_min`).
  *Примечание:* старые листы «Выгрузка»/«Фиксированное время» оставлены как в
  v0.2 (там ночные смены по-прежнему сырые); корректные часы — в листах по
  отделам и аналитике.
- **Обед** — вычитается пересечение обеденного окна графика с интервалом
  [вход, выход]. Пришёл после окна — не вычитается.
- **% отработано** — отпуск/больничный/командировка **засчитываются** в норму
  (отпускник за весь месяц ≈ 100%, а не «недоработал»).
- **Опоздания** — приход позже «Начало смены»; **переработки** — сверх «Длит.смены».

## Новые листы в «Общая выгрузка.xlsx»
- **по отделам/кабинетам** (`Отдел …`, `Отдел - Кабинет`) — табель, разбитый
  по отделам; большие отделы дробятся по кабинетам. С колонкой «Обед».
- **Отклонения** — только спорные записи для ручной проверки: нет входа/выхода,
  отметка лишь в одной системе (для dual-tracked / «Контроль ЛЭЗ»),
  неправдоподобные часы, выход с территории > 30 мин (ЛЭЗ, с указанием
  фактической длительности отлучки), расхождение времён. Отлучки ищутся
  только ВНУТРИ смены, поэтому межсменные перерывы ночников не флагуются.
- **Бухгалтерия (аванс)** — все сотрудники по доле отработанного, группы
  <25 / 25–50 / 50–75 / >75 %, у кого <50 % — красным.
- **Нормы** — отработано + зачёт отсутствий против нормы периода, столбец «+/-».
- **Опоздания и переработки** — свод по сотруднику за период.

## Кросс-проверка двух систем
«Отметка во внутренней системе есть, а в ЛЭЗ нет» помечается как подозрительная
(кандидат на «отметился за отсутствующего») для сотрудников, которые реально
отслеживаются обеими системами (определяется автоматически) **или** помечены
«Контроль ЛЭЗ = да». Цеховых (только ЛЭЗ) и офисных (только внутренняя) это не
заливает ложными срабатываниями.

## Запуск
- Как раньше: запустить скрипт, выбрать папку.
- Без диалогов (автоматический режим): `python legacy/_run.py "legacy/SCUD(fixed_time)_v0.3.py" "<папка>"`.

## Служебные файлы (для разработки/проверки, в `legacy/`)
`legacy/_run.py` — запуск без GUI; `legacy/_cmp.py` — сравнение xlsx;
`legacy/_test_compute.py`, `legacy/_test_phase4.py` — тесты логики;
`legacy/_demo_setup.py` — генератор демо-данных.
Файлы `_backup/` можно удалить (резервные копии исходных выгрузок).
