# Инструкция: настройка отделов, обедов, графиков и параметров

Все настроечные файлы лежат в папке **`ЛЭЗ\`** рядом с данными. Программа
создаёт их пустыми шаблонами при первом запуске — вам нужно их заполнить и
запустить ещё раз. Повторный запуск **не затирает** заполненное: в справочник
сотрудников только дописываются новые ФИО, в Настройки.json — только новые
параметры.

## Главный принцип (как всё связано)

```
Справочник_сотрудников.xlsx        Графики_нормы.xlsx
  ФИО ──ставите──► Отдел             График ──► Норма по месяцам
                   График  ─────────► (та же метка)  Начало смены
                   Контроль ЛЭЗ                       Длит. смены
                                                      Обед нач / Обед кон
```

То есть: **обед, нормы, начало и длительность смены задаются НЕ для каждого
сотрудника, а для ГРАФИКА.** Сотруднику вы лишь присваиваете метку графика.
Графиков обычно немного (2–5), сотрудников — сотни. Это и экономит время.

## Шаг 1. Отделы и кабинеты — файл `Справочник_сотрудников.xlsx`

Колонки: **ФИО | Отдел | Кабинет | График | Фикс.время | Контроль ЛЭЗ**

ФИО уже вписаны программой (все, кто встречался в данных). Заполняете остальное.
*(Если колонки «Кабинет» в вашем файле ещё нет — она появится сама при
следующем запуске; данные не теряются.)*

1. Откройте файл в Excel.
2. В колонке **Отдел** напишите название отдела напротив каждого ФИО.
   Одинаковый текст = один лист в табеле. Примеры: `Цех 1`, `Производство`,
   `Бухгалтерия`. Кого не заполнили — попадут на лист «Отдел Без отдела».
3. В колонке **Кабинет** (необязательно) укажите кабинет — для **больших
   отделов**, которые нецелесообразно держать одним списком. Тогда отдел
   дробится на отдельные листы по кабинетам:
   - заполнен и Отдел, и Кабинет → лист **«Отдел - Кабинет»** (напр.
     «Производство - Каб 5»);
   - заполнен только Отдел → один лист **«Отдел …»** на весь отдел;
   - в одном отделе можно часть людей развести по кабинетам, часть оставить
     без кабинета (последние попадут на общий лист отдела).
4. Быстро на всех сразу:
   - Отсортируйте лист (Данные → Сортировка), заполняйте блоками, протягивая
     ячейку вниз (за правый нижний угол).
   - Или проставьте отделы/кабинеты в отдельном столбце и вставьте обратно.

| ФИО | Отдел | Кабинет | График | Фикс.время | Контроль ЛЭЗ |
|---|---|---|---|---|---|
| Иванов Иван Иванович | Производство | Каб 5 | Смена12 | | да |
| Сидоров Иван Петрович | Производство | Каб 6 | Смена12 | | да |
| Петров Пётр Петрович | Бухгалтерия | | День5/2 | 08:00 | нет |

- **Фикс.время** — необязательно. Если стоит (например `07:40`) и человек пришёл
  раньше — в табель подставится это время. Мигрирует из старого Сотрудники.txt.
- **Контроль ЛЭЗ** — `да`, если человек обязан проходить заводскую проходную.
  Тогда «есть отметка во внутренней системе, но нет в ЛЭЗ» попадёт в отклонения
  (кандидат на «отметился за отсутствующего»). Для офисных, не ходящих через
  проходную, ставьте `нет`.

## Шаг 2. Графики, обед, начало и длительность смены — файл `Графики_нормы.xlsx`

Колонки: **График | Месяц | Норма | Начало смены | Длит.смены | Обед нач | Обед кон**

Одна строка = один график за один месяц. **Удалите строку-пример `ПРИМЕР_5x2`.**

1. Придумайте метки графиков — ровно те же, что ставите в Справочнике
   (регистр и пробелы должны совпадать). Например `День5/2` и `Смена12`.
2. На каждый график заполните **по 12 строк — на каждый месяц года** (норма
   своя по месяцам, остальное обычно одинаково — протяните вниз):

| График | Месяц | Норма | Начало смены | Длит.смены | Обед нач | Обед кон |
|---|---|---|---|---|---|---|
| День5/2 | 2026-01 | 136 | 08:00 | 8 | 12:00 | 12:30 |
| День5/2 | 2026-02 | 151 | 08:00 | 8 | 12:00 | 12:30 |
| День5/2 | 2026-03 | 168 | 08:00 | 8 | 12:00 | 12:30 |
| … | … | … | … | … | … | … |
| Смена12 | 2026-01 | 160 | 08:00 | 11 | 12:00 | 12:30 |
| Смена12 | 2026-02 | 159 | 08:00 | 11 | 13:00 | 13:30 |

Пояснения к колонкам:
- **Месяц** — формат `ГГГГ-ММ` (например `2026-04`). Норму берёте из своего
  производственного календаря по графику.
- **Норма** — часы за месяц (число).
- **Начало смены** — `ЧЧ:ММ`. По нему считаются **опоздания** (пришёл позже —
  попадёт в «Опоздания и переработки»).
- **Длит.смены** — часы за смену (число). По ней считается **переработка**
  (отработал больше — попадёт туда же).
- **Обед нач / Обед кон** — окно обеда `ЧЧ:ММ`. Из отработанного вычитается
  только пересечение этого окна с интервалом [приход, уход]. Пришёл позже окна —
  не вычитается ничего.

> Начало/длительность/обед берутся по графику (последняя строка графика). Если
> у графика обед один и тот же весь год — просто продублируйте значения во всех
> 12 строках.

## Шаг 3. Обед «для всех сотрудников»

Отдельной настройки обеда по сотрудникам нет — обед задаётся **через график**:
1. Если у всех одинаковый обед — заведите один график (или несколько по
   нормам), укажите в нём окно обеда, и присвойте этот график всем в Справочнике.
2. Если обед разный (например у цеха 12:00–12:30, у офиса 13:00–13:45) — заведите
   разные графики с разными окнами обеда и присвойте кому какой.

---

## Шаг 4. Нормы на год вперёд

Это те же строки из Шага 2: на каждый график — 12 строк (по месяцам) с нормой.
Лист **«Нормы»** в табеле автоматически посчитает по каждому сотруднику:
отработано + зачёт отсутствий − норма за период (столбец «+/-»). Отпуск,
больничный, командировка идут **в зачёт** нормы.

---

## Шаг 5. Отсутствия и командировки

- **`Отсутствия.xlsx`** (от бухгалтерии): **ФИО | Тип | Дата с | Дата по**.
  Тип — `отпуск` или `больничный`. Даты — `ДД.ММ.ГГГГ`.
- **`Командировки.xlsx`**: **ФИО | Дата с | Дата по** (тип проставится сам).

Эти дни не попадут в отклонения как «нет отметки» и зачтутся в норму.

| ФИО | Тип | Дата с | Дата по |
|---|---|---|---|
| Иванов Иван Иванович | отпуск | 01.04.2026 | 14.04.2026 |

## Шаг 6. Пороги — файл `Настройки.json`

Менять не обязательно. Значения по умолчанию:

| Параметр | Смысл | Умолч. |
|---|---|---|
| `reentry_gap_min` | отлучка за проходную больше N минут → в отклонения | 30 |
| `lateness_grace_min` | «прощаем» опоздание до N минут | 0 |
| `shift_gap_min` | перерыв больше N минут начинает новую смену (для ночных) | 300 |
| `max_shift_min` | максимальная длительность смены, минут | 960 |
| `time_mismatch_min` | расхождение времён ЛЭЗ/внутренней системы для флага | 120 |
| `implausible_hours_max` | больше N часов за смену → подозрительно | 16 |

Формат — обычный JSON, например: `"reentry_gap_min": 45`.

## Шаг 7. Запуск и проверка

1. Сохраните все файлы (закройте их в Excel, иначе будет «доступ запрещён»).
2. Запустите табель (см. README_v0.3.md). 
3. Проверьте в `Общая выгрузка.xlsx`:
   - листы **Отдел …** — по вашим отделам, с колонкой «Обед»;
   - **Бухгалтерия (аванс)** — группы по %, у кого <50 % — красным;
   - **Нормы** — столбец «+/-»;
   - **Опоздания и переработки** — должны появиться, раз заданы графики.

Если что-то не делится по отделам или показывает «нет нормы» — значит у этих
сотрудников не заполнены **Отдел** или **График**, либо метка графика в
Справочнике не совпадает с меткой в Графики_нормы (проверьте пробелы/регистр).

## Минимальный быстрый старт
1. В `Справочник_сотрудников.xlsx`: всем — Отдел и одну метку графика (напр. `Осн`).
2. В `Графики_нормы.xlsx`: удалить пример, завести график `Осн` на 12 месяцев
   (норма, начало смены, длит., обед).
3. Запустить. Дальше можно дробить графики/отделы по мере необходимости.
