# Табель СКУД

Автоматическое построение табеля рабочего времени из выгрузок систем контроля
и управления доступом (СКУД). Программа сводит отметки нескольких независимых
систем доступа в единый табель, считает отработанные часы (с учётом обеда,
ночных смен, норм и отсутствий) и формирует аналитические листы Excel для
кадров и бухгалтерии.

Актуальная версия CLI — **v0.3** (`legacy/SCUD(fixed_time)_v0.3.py`). Она полностью
обратно совместима с v0.2: старые листы выгрузки остаются без изменений,
новая аналитика добавляется отдельными листами.

---

## Что делает программа

1. Читает сырые выгрузки из систем доступа (см. «Источники данных»).
2. Нормализует события входов/выходов по каждому сотруднику и каждому дню.
3. Разбивает события на смены (в т.ч. ночные, через полночь).
4. Считает по каждому дню: вход/выход, отработанные часы, вычет обеда,
   опоздания, переработки.
5. Сводит по сотруднику за период: отработано, зачёт отсутствий, % к норме.
6. Кросс-проверяет внутренние СКУД против заводской проходной (ЛЭЗ), чтобы
   выявить подозрительные отметки.
7. Пишет результат в Excel: классический табель + аналитические листы.

## Источники данных

| Система | Роль | Формат |
|---|---|---|
| **StorK** | внутренняя СКУД (кабинеты/здания) | `StorK.csv` |
| **SIGUR** | внутренняя СКУД (кабинеты/здания) | `SIGUR.xlsx` |
| **Hikvision** | внутренняя СКУД (опционально) | xlsx |
| **ЛЭЗ** | проходная завода (периметр) | `ЛЭЗ/lez.xlsx` |

**Ключевая идея архитектуры:** отметки внутренних СКУД и проходной ЛЭЗ
хранятся и обрабатываются **раздельно**. Это позволяет кросс-проверять две
системы и ловить случаи «отметился за себя и за отсутствующего» во внутренней
системе (есть отметка внутри, но человек не проходил проходную).

- **ЛЭЗ** = проходная завода (периметр).
- **StorK + SIGUR (+ Hikvision)** = внутренние СКУД (кабинеты/здания).
- В v0.2 ЛЭЗ была лишь резервом; в v0.3 это отдельный источник для сверки.

## Справочники (папка `ЛЭЗ/`)

При первом запуске программа создаёт пустые шаблоны. Повторный запуск **не
затирает** заполненное — только дописывает новые ФИО и новые параметры.

| Файл | Что задаёт |
|---|---|
| `Справочник_сотрудников.xlsx` | ФИО, **Отдел**, **Кабинет**, **График**, **Фикс.время**, **Контроль ЛЭЗ**. ФИО дописываются автоматически. |
| `Графики_нормы.xlsx` | По каждому графику и месяцу: **Норма** (часы), **Начало смены**, **Длит.смены**, **Обед нач**, **Обед кон**. |
| `Отсутствия.xlsx` | Выгрузка бухгалтерии: ФИО, Тип (отпуск/больничный), Дата с, Дата по. |
| `Командировки.xlsx` | ФИО, Дата с, Дата по. |
| `Настройки.json` | Пороги отклонений и разбиения смен. |
| `Сотрудники.txt` | Легаси-список сотрудников и фикс. времени (`ФИО=ЧЧ:ММ`). |

**Главный принцип:** обед, нормы, начало и длительность смены задаются **не для
сотрудника, а для графика**. Сотруднику присваивается только метка графика —
графиков обычно немного (2–5), сотрудников сотни.

> Подробная пошаговая инструкция по заполнению — в [НАСТРОЙКА.md](НАСТРОЙКА.md).
> Обзор возможностей v0.3 — в [README_v0.3.md](README_v0.3.md).

## Что считается (логика v0.3)

- **Ночные/сменные смены.** События разбиваются на смены по времени: перерыв
  больше `shift_gap_min` (по умолчанию 5 ч) начинает новую смену. Поэтому смена
  «вход 20:00 → выход 07:30 следующего дня» считается как ~11.5 ч, а не как
  ноль/минус. Внутрисменный обед/короткая отлучка сливаются в одну смену.
- **Обед.** Из отработанного вычитается только пересечение обеденного окна
  графика с интервалом [приход, уход]. Пришёл после окна — не вычитается.
- **Нормы и % отработано.** Отпуск, больничный, командировка **засчитываются** в
  норму (отпускник за месяц ≈ 100 %, а не «недоработал»). Метка <50 % —
  недоработка без причины.
- **Опоздания** — приход позже «Начала смены»; **переработки** — сверх
  «Длительности смены».
- **Фиксированное время.** Если у сотрудника задано фикс. время прихода и он
  пришёл раньше — в табель подставляется фикс. время; позже — фактическое.

### Кросс-проверка двух систем
«Есть отметка во внутренней системе, но нет в ЛЭЗ» помечается как подозрительная
(кандидат на «отметился за отсутствующего») — но только для сотрудников,
реально отслеживаемых обеими системами (определяется автоматически) **или**
помеченных «Контроль ЛЭЗ = да». Цеховых (только ЛЭЗ) и офисных (только
внутренняя) ложными срабатываниями не заливает.

## Выходные листы (`Общая выгрузка.xlsx`)

Помимо классических листов v0.2 («Выгрузка», «Фиксированное время», «Полная
выгрузка»), добавляются:

- **По отделам/кабинетам** — табель по отделам; большие отделы дробятся по
  кабинетам, с колонкой «Обед».
- **Отклонения** — только спорные записи для ручной проверки: нет входа/выхода,
  отметка лишь в одной системе, неправдоподобные часы, выход с территории сверх
  порога, расхождение времён систем.
- **Бухгалтерия (аванс)** — сотрудники по доле отработанного (группы
  <25 / 25–50 / 50–75 / >75 %; у кого <50 % — красным).
- **Нормы** — отработано + зачёт отсутствий против нормы периода, столбец «+/-».
- **Опоздания и переработки** — свод по сотруднику за период.

## Запуск

```bash
# Обычный режим (диалог выбора папки):
python "legacy/SCUD(fixed_time)_v0.3.py"

# Автоматический режим без GUI (для скриптов/проверки):
python legacy/_run.py "legacy/SCUD(fixed_time)_v0.3.py" "<путь к папке с данными>"
```

> Перед запуском закройте справочники и выгрузки в Excel — иначе запись упадёт
> с «доступ запрещён» (PermissionError ловится, но новые колонки/ФИО не
> допишутся).

### Требования
- Python 3.12
- `pandas` 3.0, `openpyxl`, `xlsxwriter`

## Замечание о данных

Репозиторий содержит **только исходный код и документацию**. Реальные выгрузки
СКУД и заполненные справочники содержат персональные данные сотрудников (ФИО,
отделы, время прохода) и **в репозиторий не выкладываются**. Перед добавлением
рабочих файлов в git настройте `.gitignore`, исключающий `*.xlsx`, `*.csv`,
`ЛЭЗ/` и `_backup/`.
